								| /home/index?x=10 - URL
GET /home/index?x=10  HTTP/1.1	| /home/index - path
Host: localhost					| ?x=10 - Query
Connection: close				| Host, Connection - заголовки (headers)
								| Ключ: значення\r\n
								| HTTP/1.1 - протокол (scheme)
Структура відповіді				| 
								| 
HTTP/1.1 200 OK					| 200 - response status code
Connection: close				| OK - reason phrase (status message)
Content-Type: text/html			| Заголовки
Content-Length: 100500			| 
								| порожний рядок (\r\n)
<!doctype html>......			| Тіло (body) - до кінця пакету

-------------------------------------------------------------------------
Особливості:
Метод запиту - перше "слово" у пакеті, є 9 стандартних методів, також
існують позастандартні, більшість серверів не обмежують нестандартні методи.

Заголовки: більшість універсальні, але деякі мають спец. призначення
- тільки у запиті
- тільки у відповіді
- тільки з відповідним статус-кодом
  (наприклад, при статусі 401 "Unauthorized" має бути посилання на авторизацію,
   при статусі 301\302 "Redirect" - має бути нова адреса Location)

Тіло може бути як у запиті, так і у відповіді. За винятками:
запити методами GET та HEAD не повинні мати тіло. Цей факт браузер блокує.
